# RB-002 — PostgreSQL Down

**Alert:** `PostgresDown`  **Severity:** CRITICAL  **MTTR target:** < 15 min

## Immediate triage

```bash
docker ps --filter name=aegis-pg
docker logs aegis-pg-primary --tail 50
docker exec aegis-pg-primary pg_isready -U aegis -d aegis
```

## Decision tree

**Container not running:**
```bash
docker compose up -d postgres-primary
# Wait ~20s for healthcheck
docker compose ps postgres-primary
```

**OOM:**
```bash
docker stats aegis-pg-primary --no-stream
# Increase shared_buffers or container memory limit
```

**Disk full:**
```bash
docker exec aegis-pg-primary df -h /var/lib/postgresql
# Prune old WAL / run VACUUM
docker exec aegis-pg-primary psql -U aegis -d aegis -c 'VACUUM ANALYZE;'
```

**Corrupt data:**
```bash
# Check pg_controldata
docker exec aegis-pg-primary pg_controldata | grep "Database cluster state"
# If not "in production": restore from backup — see DR section below
```

## Point-in-time recovery

```bash
# Stop C2 to prevent writes during recovery
docker compose stop c2-1 c2-2 scheduler

# Restore from latest pg_basebackup
docker run --rm -v postgres-primary-data:/target \
  -v /backups:/backups \
  postgres:16-alpine \
  pg_restore -d postgres://aegis:$PW@/aegis /backups/latest.dump

# Restart
docker compose start postgres-primary c2-1 c2-2 scheduler
```

# RB-003 — Redis Down

**Alert:** `RedisDown`  **Severity:** WARNING  **MTTR target:** < 10 min

C2 continues to function (circuit breaker degrades gracefully). SSE pub/sub
falls back to in-process only — operators on multiple C2 instances won't see
each other's events until Redis is restored.

## Steps

```bash
docker compose restart redis-master
# Check sentinel quorum (HA profile)
docker exec aegis-redis-sentinel-1 redis-cli -p 26379 sentinel masters
```

If Sentinel has promoted a new master:
```bash
# Get new master address from sentinel
NEW_MASTER=$(docker exec aegis-redis-sentinel-1 \
  redis-cli -p 26379 sentinel get-master-addr-by-name aegis-master | head -1)
# Update C2 REDIS_HOST env and restart
```

# RB-004 — High API Latency / Error Rate

**Alert:** `HighApiLatency` or `HighErrorRate`  **Severity:** WARNING

## Diagnosis

```bash
# Check which paths are slow
curl -s http://localhost:5000/metrics | grep http_request_duration

# Check DB pool wait
curl -s http://localhost:5000/metrics | grep db_pool_wait

# Check circuit breaker state
curl -s -H "X-Aegis-Key: $KEY" http://localhost:5000/api/status | jq .circuit_breakers
```

## Common causes

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| DB pool wait > 5 | Slow queries / pool exhausted | Increase PG_POOL_MAX; check pg_stat_activity |
| Redis CB open | Redis timeout | See RB-003 |
| All paths slow | CPU contention | Scale c2-2; check `docker stats` |
| Specific path slow | Missing index | EXPLAIN ANALYZE the query; add index |

```bash
# Find slow queries in PostgreSQL
docker exec aegis-pg-primary psql -U aegis -d aegis -c \
  "SELECT query, calls, mean_exec_time FROM pg_stat_statements
   ORDER BY mean_exec_time DESC LIMIT 10;"
```

# RB-005 — Authentication Incident (Brute-Force)

**Alert:** `AuthBruteForce`  **Severity:** CRITICAL

## Immediate actions

```bash
# 1. Identify the offending IP from logs
docker logs aegis-c2-1 --tail 500 | grep '"action":"login_fail"' | \
  python3 -c "import sys,json,collections; \
  lines=[json.loads(l) for l in sys.stdin if '{' in l]; \
  ips=collections.Counter(l.get('ip','') for l in lines if l.get('action')=='login_fail'); \
  [print(c,ip) for ip,c in ips.most_common(10)]"

# 2. Block IP at host firewall (Linux)
sudo iptables -I INPUT -s <ATTACKER_IP> -j DROP

# 3. Check if any operators were locked out
curl -s -H "Authorization: Bearer $ADMIN_TOKEN" \
  http://localhost:5000/api/admin/operators | \
  jq '.operators[] | select(.failed_logins > 0)'

# 4. Unlock legitimate operators if needed
curl -s -X PATCH -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  http://localhost:5000/api/admin/operators/r00t_handler/role \
  -d '{"role":"admin"}'

# 5. Rotate the operator key if compromise is suspected
curl -s -X POST -H "Authorization: Bearer $ADMIN_TOKEN" \
  http://localhost:5000/api/admin/secrets/rotate/jwt
```

## If key was compromised

1. Rotate JWT secret immediately (above)
2. Revoke all sessions: `POST /api/admin/operators/{handle}/sessions/revoke`
3. Issue new operator keys to all team members out-of-band
4. Audit the `operator_audit` table for suspicious actions
5. Check `webhook_deliveries` for unexpected external calls

# RB-006 — Circuit Breaker Tripped

**Alert:** `CircuitBreakerOpen`  **Severity:** WARNING

## Check which service

```bash
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:5000/api/status | jq '.circuit_breakers'
```

## Service-specific actions

- **redis**: See RB-003
- **postgres**: See RB-002
- **nvd_api**: NVD sync is failing — not operational. Dismiss if OK.

## Manual reset (after fixing the underlying issue)

The circuit breaker resets automatically after `recovery_timeout` seconds
when a probe succeeds. To reset immediately:

```bash
docker exec -it aegis-c2-1 python3 -c "
from c2.resilience.circuit_breaker import redis_breaker, postgres_breaker
redis_breaker.reset()
postgres_breaker.reset()
print('Breakers reset')
"
```
